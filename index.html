搭建Hexo博客

一、为什么选择Hexo来搭建博客

其实开始想要搭建个人博客时，有很多种选择，不过想要免费搭建博客的话，大多数人都是用jeKyll或者Hexo + Github。百度了一下，觉得Hexo的环境配置比较简单，所以就用Hexo + Github来搭建博客了。

难道就是因为配置环境比较简单所以就选择Hexo了吗？我之前在搭建PHP环境的时候一开始没用集成包，想要自己搭建，然后过了两个星期差点放弃了PHP。个人认为，对于一个新手来说如果在搭建环境的时候浪费太多时间，会大大降低这一方面的兴趣，甚至会放弃。(我没用过jeKyll，所以不知道它的环境搭建到底难不难，用jeKyll的哥们别喷我啊)

给你们官网，自己决定：
Hexo官网：https://hexo.io/
jeKyll官网：https://www.jekyll.com.cn/

二、搭建过程

其实也没什么好说的，在简书上看到一个帖子，根据他的指导一步一步来的，也不算太难，博主比较懒，就不写教程了，人家的那个就挺好。搭建之前要先有个Github的个人主页。

Github搭建个人主页：https://pages.github.com/

搭建博客：https://www.jianshu.com/p/c9295bacd98b?utm_campaign=maleskine&utm_content=note&utm_medium=seo_notes&utm_source=recommendation

写在开始——路漫漫其修远兮

post @ 2018-01-12

路漫漫其修远兮，吾将…

第一次看见这句话的时候是在小学，看到有一本小说上面的主人公叫漫漫，是她爷爷给她起的，就是引用的“路漫漫其修远兮，吾将上下而求索”这句话。当时只知道是屈原写的。

第二次看到这句话是高中的时候，在屈原的《离骚》中看到了这句话。

第三次见到是在一个大佬的博客上。当时正在搞易班的API，因为是新手（现在依然是新手，一直在入门），所以搞得头皮发麻，百度一些没见过的PHP语句的时候，进入到一个大佬的博客，他的主页上面有一句话——路漫漫其修远兮。当时突然就读懂了这句话，觉得这句话很牛逼。所以我的博客主页上面也是这句话。

编程也好，健身也好，不管做什么，都是一条漫长的路，不可急于求成，功到自然成。这句话也送给正在看的你共勉。

路漫漫其修远兮，后面的半句我还没资格说。

中文测试

post @ 2017-12-26

第一次搭建博客，搞了一周多终于要写一篇博文测试一下了，顺便学习一下Markdown的基本语法。(start:2017-12-26 08:46:37)

一、区块元素

1.段落和换行

一个 Markdown 段落是由一个或多个连续的文本行组成，它的前后要有一个以上的空行（空行的定义是显示上看起来像是空的，便会被视为空行。比方说，若某一行只包含空格和制表符，则该行也会被视为空行）。普通段落不该用空格或制表符来缩进。

「由一个或多个连续的文本行组成」这句话其实暗示了 Markdown 允许段落内的强迫换行（插入换行符），这个特性和其他大部分的 text-to-HTML 格式不一样（包括 Movable Type 的「Convert Line Breaks」选项），其它的格式会把每个换行符都转成
标签。

如果你确实想要依赖 Markdown 来插入
标签的话，在插入处先按入两个以上的空格然后回车。
